Comandos e conceitos básicos do git

Git básico

O que é o Git?

O Git é um sistema de controle de versão distribuído utilizado para registrar o histórico de edições de qualquer tipo de arquivo.

Ciclo de vida dos arquivos no Git

Untracked

Ao criar um novo arquivo em um projeto gerenciado pelo Git os arquivos iniciam como untracked, ou seja, o mesmo ainda não foi adicionado ao monitoramento do Git.

Tracked

Quando adicionamos arquivos ao monitoramento do Git, o status desse arquivo passa a ser tracked. A partir de então o Git passa a controlar as mudanças neste arquivo. Os arquivos com esse estado são conhecidos como new file.

Modified

Quando modificamos um arquivo que está sendo rastreado (no estado tracked), seu estado passa a ser modified.

Staged

Quando um arquivo está pronto para ser comitado (está no estado tracked e com tudo finalizado) então está pronto para ser enviado para o repositório remoto (Github) com um commit.

Observação: Apenas um arquivo no estado tracked pode ir para staged. Toda vez que um arquivo estiver no estado modified é necessário modificá-lo para tracked.

Ignorar arquivos

.gitignore

Para que o Git ignore arquivos ou diretórios ao fazer um commit, basta criar um arquivo chamado .gitignore na raiz do seu repositório local e escrever o nome dos arquivos a serem ignorados

Dica

Para ignorar todos os arquivos com uma extensão em comum:

*.nomeDaExtensão

Para ignorar uma pasta

**nomeDaPasta

Operações básicas no terminal

Inicializar o git no seu repositório

Basta digitar no terminal:
git init

Configurações básicas após criar um repositório


git config user.name "seu nome"
git config user.email "seu email"

Adicionando um arquivo ao estado tracked

git add nomeDoArquivo git add . O último adiciona todos os itens de uma vez para tracked.

Realizando um commit

git commit -m "mensagem"

Log de edição do repositório

git log git log nMostra os últimos n logs git log --onelineVersão simplificada dos logs

Indo para logs antigos

git checkout idDoLog

Voltando para o log mais recente

git checkout master

Renomeando arquivo

git mv nomeAntigo nomeNovo

Comparando arquivos

Quando um arquivo está no estado staged, com o comando get diff --staged É possível ver as diferenças entre o que está prestes a ser comitado e o que foi comitado por último

Corrigindo mensagem do último commit

Mensagem errada: atualizacaoo de bugs corrigindo: git commit --amend -m "Atualização de bugs"

Implementando algo que era pra estar no último commit feito

git commit --amend -m "mensagem" Esse comando adiciona tudo no mesmo commit

Resetando arquivos modified para o último commit

git reset HEAD --hard

Para resetar apenas um arquivo ao invés de todo o repositório

git checkout nomeDoArquivo

Descartando o último commit

git resert HEAD^ --hard

Git intermediário

Branches

Branch nada mais é do que uma ramificação do projeto. Branch master é o ramo principal do seu projeto

Retornando as branches do projeto

git branch

Criando uma branch

git branch nomeDaBranch

Criando uma branch e já acessando-a

git checkout -b nomrDaBranch

Mudando de branch

git checkout nomeDaBranck

Voltando para a branch master

git chekcout master

Deletando uma branch

git branch -d nomeDaBranch

Merge

O merge é utilizado para combinar dois branches e juntar tudo num novo commit. Como efetuar um merge:na branch master, basta digitar no terminal: git merge nomeDaBranch

Rebase

Ao contrário do merge que efetua o commit no final da branch master, o rebase adiciona o commit no começo da branchgit rebase nomeDaBranch

Clone e Push

Clonando um repositório

Para clonar um repositório para a sua máquina basta digitar no terminalgit clone link/CaminhoDoRepositorio

Push

Para submeter mudanças e/ou novos arquivos para um repositorio central, basta digitar no terminalgit push

Fetch e Pull

Fetch

Fetch baixa as atualizações de um repositório remoto mas não as aplica no repositório local, ou seja, não faz o merge.git fetchPara efetuar um merge e atualizar o repositorio git rebase

Pull

O comando pull baixa as atualizações do repositório remoto e já as aplica, ou seja, já efetua o mergegit pull

Trabalhando com tags

Tag

Tags demarcam um ponto (commit) que representa alguma mudança significativa no código, ou seja, uma versão do projeto. Para criar uma tag: git tag nomeDaTag

Submetendo a tag ao repositório remoto

git push origin nomeDaTag
